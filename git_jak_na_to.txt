Super! To je skvělé, že jsi si vše doplnil do nové větve na GitHubu. Teď si ukážeme, jak tuto větev dostaneš do svého počítače.
Předpokládám, že máš na svém počítači již naklonované (stažené) hlavní úložiště (repozitář) a chceš do něj přidat novou větev.
Jak dostat novou větev z GitHubu do počítače
Postup je jednoduchý a zahrnuje dva hlavní kroky: načtení změn a přepnutí na větev.
Krok 1: Načti nejnovější změny z GitHubu (Fetch)
Nejprve musíš svému lokálnímu Git repozitáři říct, aby si stáhl informace o všech nových větvích a změnách z GitHubu.
1.	Otevři terminál nebo Git Bash: Přejdi do složky, kde máš naklonovaný svůj projekt.
o	Ve Windows můžeš kliknout pravým tlačítkem myši ve složce projektu a vybrat "Git Bash Here" nebo "Open in Terminal".
o	V macOS/Linux otevři terminál a použij příkaz cd cesta/k/tvemu/projektu.
2.	Načti změny: Zadej tento příkaz:
Bash
git fetch origin
o	git fetch: Říká Gitu, aby stáhl nejnovější informace z vzdáleného úložiště.
o	origin: Je standardní název pro tvé vzdálené úložiště na GitHubu.
Tento příkaz stáhne metadata o nových větvích, ale nepřepne tě na ně automaticky a nezmění soubory ve tvém pracovním adresáři. Pouze aktualizuje tvůj lokální přehled o vzdáleném repozitáři.
Krok 2: Přepni na novou větev (Checkout)
Jakmile jsi načetl informace o nové větvi, můžeš se na ni lokálně přepnout.
1.	Přepni se na větev: Zadej tento příkaz, kde nazev-tve-nove-vetve nahradíš skutečným názvem tvojí větve na GitHubu (např. feature/player-name-and-score):
Bash
git checkout nazev-tve-nove-vetve
o	git checkout: Slouží k přepínání mezi větvemi.
o	Pokud větev existuje pouze vzdáleně (na GitHubu) a ještě jsi na ni lokálně nepřepínal, Git automaticky vytvoří lokální větev se stejným názvem a nastaví ji tak, aby sledovala vzdálenou větev. V takovém případě uvidíš zprávu podobnou: Branch 'nazev-tve-nove-vetve' set up to track remote branch 'nazev-tve-nove-vetve' from 'origin'. Switched to a new branch 'nazev-tve-nove-vetve'
A to je vše! Teď bys měl mít všechny soubory z nové větve ve své pracovní složce a můžeš na nich začít pracovat.
Co dělat dál?
•	Pravidelná aktualizace: Pokud na té větvi budeš pracovat dál a nahrávat změny na GitHub, nezapomeň před každým dalším vývojem na svém PC provést git pull (který je kombinací git fetch a git merge), abys získal nejnovější verzi:
Bash
git pull origin nazev-tve-nove-vetve
•	Sloučení (Merge): Až bude větev hotová a otestovaná, budeš ji chtít sloučit do hlavní větve (např. main nebo master). To se obvykle dělá přes Pull Request na GitHubu, ale lze to udělat i lokálně:
Bash
git checkout main        # Přepneš se na hlavní větev
git pull origin main     # Získáš nejnovější změny z hlavní větve
git merge nazev-tve-nove-vetve # Sloučíš svou feature větev do hlavní
git push origin main     # Odešleš sloučené změny na GitHub
Pokud bys měl/a jakékoli potíže nebo dotazy k těmto příkazům, dej vědět!
